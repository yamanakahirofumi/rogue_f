# ロギング方針

デバッグおよび保守のため、適切なロギングを行います。

## 1. 基本方針
本プロジェクトでは、ブラウザの標準的な `console` API を使用して、アプリケーションの動作状況や例外発生時の情報を記録します。

## 2. ログレベルの使い分け
ログレベルは以下のように使い分けます。

- **console.log (INFO相当)**：
    - アプリケーションの主要なイベント（画面遷移、リソース読み込み成功、データの更新など）。
    - 正常系の重要な進行状況を記録します。
- **console.warn (WARNING相当)**：
    - 続行は可能だが注意が必要な事象（非推奨な関数の使用、ユーザーによる不適切な操作など）。
- **console.error (ERROR相当)**：
    - 処理の続行が困難な予期せぬ実行時エラー、API 通信の失敗、必須リソースの読み込み失敗。

## 3. 環境別制御
- **開発環境 (Development)**：
    - すべてのログレベルを出力します。詳細なデバッグ情報も含まれます。
- **本番環境 (Production)**：
    - 基本的に `console.log` は抑制し、`console.error` および重要な `console.warn` のみを出力するように構成します。

## 4. 出力内容
- メッセージ（何が起きたか）
- 関連するデータオブジェクト（必要に応じて）
- エラー発生時は例外オブジェクト（スタックトレースを含む）
