# Optimization-Strategy

## 1. パフォーマンス最適化方針
現在はMVPフェーズのため、特別な最適化処理は導入されていません。今後、描画負荷や通信量が増大した場合に、以下の手法の導入を検討します。

### 1.1 描画の最適化
- **描画範囲の制限（カリング）**: プレイヤーの視界外のタイル描画をスキップする。
- **Change Detection の最適化**: `OnPush` 戦略の採用などによる Angular の変更検知回数の削減。

### 1.2 通信の最適化
- **SSE のペイロード削減**: 変更があったタイルのみを送信する差分更新の強化。
- **データ圧縮**: ネットワーク帯域を節約するためのデータ転送形式の検討。
