# Warehouse-System

## 1. 概要
倉庫（ストック）システムは、管理者が自身の世界（ダンジョン）を運営するために必要なリソース（モンスター、アイテム、資材）を保管・管理するための中心的な仕組みです。探索で得た成果を蓄積し、戦略的に再配置するための基盤となります。

## 2. 倉庫の種類と機能

### 2.1 モンスター・ストック
- 捕獲した野生のモンスターや、繁殖によって生まれたモンスターを保管します。
- 保管されているモンスターは、以下の用途に使用できます。
    - 自身のダンジョンへの配置（[モンスターシステム](Monster-System.md) 参照）。
    - 繁殖の親個体としての利用（[モンスター繁殖システム](Monster-Breeding-System.md) 参照）。
- 各個体には、レベル、ステータス、継承された特性などの詳細情報が保存されます。

### 2.2 アイテム・ストック
- プレイヤーのデスペナルティによって没収したアイテムや、管理者自ら収集したアイテムを保管します。
- 保管されているアイテムは、以下の用途に使用できます。
    - ショップへの陳列・販売（[ショップシステム](Shop-System.md) 参照）。
    - ダンジョンのクリア報酬としての設定。
    - 管理者自身による装備や使用（検討中）。

### 2.3 資材・トラップ・ストック
- ダンジョン構築に必要な建築資材や、配置用のトラップを保管します。
- トラップの配置には、ここから資材を消費します（[トラップシステム](Trap-System.md) 参照）。

## 3. 容量と拡張性

### 3.1 初期容量
- モンスターおよびアイテムの保管数には初期上限が設定されています（モンスター 30 体、アイテム 100 個）。

### 3.2 倉庫の拡張
- ゴールドや特定の素材を消費することで、倉庫の容量を拡張できます。
- 拡張により、より大規模なダンジョン運営や多様なモンスターのコレクションが可能になります。

## 4. 管理画面での操作
- 管理者は専用の「倉庫管理画面」を通じて、在庫の一覧表示、ソート、検索、および各システム（ショップ、ダンジョン配置等）への移動操作を行います。
