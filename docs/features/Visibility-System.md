# Visibility-System

## 1. 概要
視界システムは、プレイヤーがダンジョン内で視認できる範囲と、それに伴うマップの探索（開拓）を制御する仕組みです。このシステムは、プレイヤーの安全確保や戦略的な移動、およびモンスターとの遭遇に直結します。

## 2. 視界の基本メカニズム

### 2.1 視界半径 (Visibility Radius)
- **基本視界半径**: プレイヤーの周囲 **5 マス** です。
- プレイヤーを中心とした円形（またはマンハッタン距離/チェビシェフ距離に基づく範囲）のエリアが、リアルタイムで視認可能となります。
- 視界内の地形、モンスター、アイテム、および発見済みのトラップが描画されます。

### 2.2 視線の遮蔽 (Line of Sight)
- **遮蔽物**: [地形マスターリスト](Terrain-Master-List.md) において「視界も遮る」と定義されている地形（例：壁 `#`、閉じている扉 `+`）は、視線を遮ります。
- **判定アルゴリズム**: 直線的な視界判定には **Bresenham's line algorithm** を使用します。
- 遮蔽物の向こう側にあるエンティティや地形は、視界半径内であっても視認することはできません。

## 3. 明かりと照明 (Lighting)

暗いダンジョン内での視界を拡張するための仕組みです。

### 3.1 照明アイテム
特定のアイテムを使用または所持することで、視界を補助できます。
- **松明 (`torch`)**:
    - 周囲 **3 マス** を照らし、壁などの遮蔽物がない限り、その範囲内の地形を常に視認可能にします。
    - 松明による照明は、プレイヤー自身の視界半径とは独立して（または累積して）計算されます。

### 3.2 恒久的な照明施設
- **回復の泉 (`&`)** や **転送門 (`O`)** などの一部の施設は、それ自体が微かな光を放っており、周囲 1 マスを常に照らしています。

## 4. フォグ・オブ・ウォー (Fog of War)

探索済みのエリアと未探索のエリアを区別する仕組みです。

### 4.1 探索済みエリア (Explored Area)
- 一度視界に捉えたエリアは「探索済み」として記録されます。
- 探索済みエリアは、現在の視界外であっても地形データ（床、壁、通路等）がミニマップやメイン画面上に（通常より暗く、または特定のスタイルで）表示され続けます。

### 4.2 リアルタイム情報
- 探索済みであっても、現在の視界外にいる **モンスター (`M`)** や **他のプレイヤー (`P`)** の動的な情報は表示されません。
- トラップ (`^`) については、一度発見していれば探索済みエリアとして表示され続けます。
