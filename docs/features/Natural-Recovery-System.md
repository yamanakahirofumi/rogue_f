# Natural-Recovery-System

## 1. 概要
自然回復システムは、時間の経過とともにプレイヤーの HP およびスタミナが徐々に回復する仕組みです。このシステムは、探索中の持続力を左右し、[満腹度システム](Hunger-System.md) や [状態異常システム](Status-Effect-System.md) と密接に連携しています。

## 2. 基本メカニズム

### 2.1 回復間隔 (Interval)
- 標準的な回復は **10 秒ごと**（1 ティック）に発生します。
- 回復のタイミングは、[IntervalService](../implementation/Implementation-Details.md#51-主要サービス) によって管理される共有タイマーに基づきます。

### 2.2 基本回復量
- **HP**: 1 ティックにつき **最大HPの 1%** 回復します。
- **スタミナ**: 1 ティックにつき **最大スタミナの 1%** 回復します。

## 3. 状態による補正
プレイヤーの現在の状態に応じて、回復量や速度が変化します。

### 3.1 満腹度の影響
[満腹度システム](Hunger-System.md) の状態に基づき、以下の補正が適用されます。

| 状態 | 満腹度の範囲 | 回復への影響 |
| :--- | :--- | :--- |
| **満腹 (Satiated)** | 80% - 100% | 自然回復速度（量）が **1.2 倍** になります。 |
| **通常 (Normal)** | 20% - 79% | 標準の回復量。 |
| **空腹 (Hungry)** | 1% - 19% | 自然回復が **完全に停止** します。 |
| **飢餓 (Starving)** | 0% | 自然回復（HP/スタミナ共に）が完全に停止し、代わりに **最大HPに基づいたダメージ** が発生します。 |

### 3.2 状態異常の影響
[状態異常システム](Status-Effect-System.md) の効果に基づき、以下の補正が適用されます。

- **再生 (Regeneration)**: 自然回復速度（量）が **2.0 倍** になります。
- **毒 (Poison)**: 毒ダメージが発生している間も、満腹度の条件を満たしていればスタミナの自然回復は行われますが、HP の自然回復は毒ダメージによって相殺または減少します。
- **飢餓 (Starving)**: 満腹度が 0%（飢餓状態）の間は、**スタミナの自然回復も完全に停止**します。

## 4. 計算ルール
- 複数の倍率補正が重なる場合、基本的には乗算で計算されます。小数点以下は**切り捨て**て適用します。
  - 例: 「満腹」かつ「再生」状態の場合: `1 (基本) * 1.2 (満腹) * 2.0 (再生) = 2.4` → 切り捨てて **2** 回復。
- 最大値（`maxHp`, `maxStamina`）を超えて回復することはありません。

## 5. 処理順序 (Processing Order)
1 ティック（10 秒）内の定期処理は、以下の順序で実行されます。これにより、毒ダメージによる死亡判定と自然回復が予測可能な形で処理されます。

1.  **継続ダメージの適用**: 毒 (Poison) や 飢餓 (Starving) による HP 減少を適用します。
2.  **死亡判定**: HP が 0 以下になった場合、即座に死亡処理を行います。
3.  **自然回復の適用**: 条件（満腹度等）を満たしている場合、HP および スタミナの自然回復を適用します。
4.  **状態異常の持続時間更新**: 全てのバフ・デバフの残り持続時間を減少させ、0 になったものを解除します。
