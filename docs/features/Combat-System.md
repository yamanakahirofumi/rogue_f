# Combat-System

## 1. 戦闘の基本コンセプト
RogueFの戦闘は、従来のターン制ローグライクとは異なり、**リアルタイム/アクティブタイム制**を採用します。プレイヤーとモンスターは、それぞれの「待機時間（クールタイム）」が経過するたびに行動が可能になります。

## 2. バトルシステム詳細

### 2.1 リアルタイム性
- 全てのエンティティ（プレイヤー、モンスター）は独立したタイマーを持ちます。
- 行動（攻撃、アイテム使用等）を行うと、その行動に応じたクールタイムが発生します。

### 2.2 攻撃と待機時間
- **アクションインターバル**: 攻撃行動には待機時間が発生します。プレイヤーの攻撃速度は「敏捷性」に依存します。詳細は **[アクションシステム](Action-System.md)** を参照してください。
- **モンスターの攻撃インターバル**: モンスターの種類により 1.5秒〜3.0秒（各モンスターの個別値については [モンスターマスターリスト](Monster-Master-List.md) を参照）。
- クールタイム（インターバル）中は次の攻撃が行えません。UI上でプログレスバー等によりクールタイムを表示します。
- 攻撃コマンドおよびレスポンスの詳細は [実装詳細](../implementation/Implementation-Details.md#22-プレイヤー情報) を参照してください。

### 2.3 ダメージ計算式
ダメージ量は以下の式をベースに算出します。

`ダメージ = (攻撃力 * 乱数(0.9〜1.1)) - (防御力 / 2)`

- 最小ダメージは 1 とします。

### 2.4 命中と回避
命中判定は、攻撃側の「器用さ」と防御側の「素早さ」を用いて算出します。

- **命中率計算式**:
    - `命中率(%) = 90 + (自身の器用さ - 敵の素早さ) * 2`
- **制約**:
    - 命中率の下限は 50%、上限は 100% とします。

### 2.5 クリティカルヒット
攻撃時に一定の確率でクリティカルヒットが発生します。

- **クリティカル発生率**: 攻撃側の「器用さ」に依存して決定されます。
    - `クリティカル発生率(%) = 器用さ * 0.5`
- **効果**: 通常よりも高いダメージ（1.5倍）を与えます。
- **判定**: `CombatResult` の `critical` フィールドによってフロントエンドに通知されます。

### 2.6 攻撃射程 (Attack Range)
攻撃が届く範囲は、装備している武器やモンスターの種類によって決まります。

- **近接攻撃 (Melee)**:
    - 射程: **1**
    - 自身の周囲 8 マスが対象となります。武器を装備していない場合や、通常の剣などの射程は 1 です。
- **遠距離攻撃 (Ranged)**:
    - 射程: **2 以上** (アイテムや個体別に定義)
    - 弓や魔法などの攻撃が該当します。
    - **視線の制約**: 攻撃者と対象の間に「壁」などの遮蔽物がある場合、射程内であっても攻撃は届きません。

## 3. モンスターAIパターン

モンスターの種類ごとの具体的な AI パターンの割り当てについては、**[モンスターマスターリスト](Monster-Master-List.md)** を参照してください。

モンスターは以下のいずれかの思考ルーチンを持ちます。

### 3.1 積極的 (Aggressive)
- プレイヤーを視界（10 マス）に捉えると、最短距離で接近し、隣接すると攻撃を繰り返します。

### 3.2 遠距離型 (Ranged)
- プレイヤーと一定の距離（4 マス）を保とうとします。
- 距離が近い場合は離れ、離れすぎている場合は近づきます。
- 射程内に入ると遠距離攻撃を行います。

### 3.3 臆病 (Cowardly)
- HPが一定割合（25%）以下になると、プレイヤーから逃げるように移動します。

### 3.4 ランダム (Random)
- 特定の目的を持たず、ランダムな方向に移動します。

## 4. 状態異常
戦闘を深めるため、スタン、毒、鈍足などの状態異常を定義します。

詳細は [状態異常システム](Status-Effect-System.md) を参照してください。
