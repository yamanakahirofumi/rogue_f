# Shop-System

## 1. 概要
ショップシステムは、管理者が自身のダンジョン内にアイテム屋を設置・運営し、プレイヤーとアイテムの売買を行うための仕組みです。世界（サーバー）内の流通量に基づいた動的な価格決定と、管理者の戦略的な価格設定を特徴とします。

## 2. ショップの種類

### 2.1 管理者ショップ (Admin Shop)
- 管理者が自身の所持品から商品を選択し、価格を設定して公開する店舗です。
- 管理者のダンジョン内に設置されます。

### 2.2 システムショップ (System Shop)
- ランダムダンジョン内に稀に出現する、自動生成されたショップです。
- 基本市場価格に基づいた標準的な価格でアイテムを販売します。

## 3. ショップ運営のメカニズム

### 3.1 商品の選定
- 管理者は自身の「ストック（[倉庫システム](Warehouse-System.md)）」から、ショップに陳列するアイテムを選択します。
- 一度に陳列できるアイテム数には上限があります（10 スロット）。

### 3.2 価格決定
- **基本市場価格 (Base Market Price)**: 世界全体の流通量（在庫数）に応じて自動計算される理論上の価格です。
  - 流通が少ないアイテムは高く、多いアイテムは安くなります。
  - **計算式**:
    - `基本市場価格 = 基本価格 * (1 + (サーキュレーション制限 / (現在の流通数 + 1)) * 0.1)`
- **販売価格の設定**: 管理者は基本市場価格を参考に、任意の価格（基本価格の 70% 〜 300%）を設定できます。

### 3.3 販売と収益
- プレイヤーがアイテムを購入すると、代金（ゴールド）は管理者の所持金に加算されます。
- 購入されたアイテムは管理者の在庫から差し引かれます。

## 4. プレイヤーの操作
- **購入**: ショップ内のアイテムを選択し、表示された価格を支払って取得します。
- **売却**: 自身のインベントリ内のアイテムをショップ（または店主）に提示し、提示された買取価格で売却します。
  - 買取価格は基本市場価格の 40% をベースに算出されます。
- **鑑定**: 店主に対し、未識別アイテムの鑑定を有料で依頼できます。詳細は [アイテム識別システム](Item-Identification-System.md) を参照してください。

## 5. 経済への影響

### 5.1 サーキュレーション制限
- [機能仕様書](Functional-Specification.md) に基づき、貴重なアイテムには世界内の存在上限が設定されています。
- ショップでの売買は、この流通カウントに即座に反映されます。

### 5.2 世界間貿易
- 他の世界から持ち込まれたアイテムがショップで売却されることで、その世界内での希少価値が変動します。
