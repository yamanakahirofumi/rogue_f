# Action-System

## 1. 概要
アクションシステムは、プレイヤーの移動、攻撃、アイテム使用などのあらゆる行動における時間的コスト（待機時間）とリソース消費（スタミナ・満腹度）を統一的に管理する仕組みです。本作のリアルタイム性を支える基幹システムです。

## 2. アクションインターバル (Action Interval)
全ての行動には、次の行動が可能になるまでの待機時間（インターバル）が発生します。

### 2.1 インターバル計算式
基本インターバルはプレイヤーの「敏捷性 (Agility)」によって短縮されます。

`アクションインターバル = 基本インターバル / (1 + 敏捷性 / 100)`

- **敏捷性 0**: インターバルは基本値のまま。
- **敏捷性 100**: インターバルは基本値の 1/2（速度2倍）。
- **敏捷性 300**: インターバルは基本値の 1/4（速度4倍）。

### 2.2 行動別基本インターバルとスタミナ消費
各アクションの基本インターバル（敏捷性 0 時）および実行時に消費されるスタミナ（ST）の一覧です。

| アクション | 基本インターバル | スタミナ消費 | 備考 |
| :--- | :--- | :--- | :--- |
| **移動 (Move)** | 0.5 秒 | 1 ST | 8方向共通。 |
| **攻撃 (Attack)** | 1.0 秒 | 2 ST | 近接・遠距離共通。 |
| **取得 (Pickup)** | 0.5 秒 | 0 ST | |
| **使用 (Use)** | 0.8 秒 | 0 ST | アイテムの使用、装備の変更。 |
| **探索 (Search)** | 1.2 秒 | 1 ST | 周囲 8 マスの隠し要素を発見。 |
| **解除 (Disarm)** | 1.2 秒 | 1 ST | トラップの解除試行。 |
| **階段 (Stairs)** | 1.0 秒 | 0 ST | 階層の移動。 |

- **スタミナ不足時の挙動**: 現在のスタミナが必要消費量に満たない状態で行動した場合、不足分のスタミナは **HP を消費**することで補われます（1 ST = 1 HP）。

## 3. 環境と状態による補正
基本インターバルに対し、地形や状態異常による乗算補正が適用されます。複数の補正がある場合は乗算で計算されます。

### 3.1 地形補正
[地形マスターリスト](Terrain-Master-List.md) に基づく補正です。

- **水 (`water`)**: インターバル **2.0 倍**
- **砂地 (`sand`)**: インターバル **1.5 倍**

### 3.2 状態異常補正
[状態異常システム](Status-Effect-System.md) に基づく補正です。

- **鈍足 (`SLOW`)**: インターバル **2.0 倍**

### 3.3 計算例
「敏捷性 100」のプレイヤーが「水」地形で「移動」する場合：
`0.5 (基本) / (1 + 100 / 100) * 2.0 (水) = 0.5 秒`

## 4. 満腹度の減少
[満腹度システム](Hunger-System.md) に基づき、アクションの種類に関わらず、1 回のアクションごとに満腹度が減少します。

- **基本減少量**: 1 アクションにつき **0.2** 減少
- **階層補正**: 階層が深くなるほど減少量が増加します。詳細は [満腹度システム](Hunger-System.md#5-ゲームバランスの調整) を参照してください。
