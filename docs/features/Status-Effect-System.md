# Status-Effect-System

## 1. 概要
状態異常システムは、プレイヤーやモンスターのステータス、行動、およびリソース管理に対して一時的または継続的な変化を与える仕組みです。戦闘の戦略性を高め、探索中のリスクと報酬のバランスを調整する重要な要素です。

## 2. 状態異常の分類
状態異常は大きく分けて「弱体化（デバフ）」と「強化（バフ）」の二種類に分類されます。

### 2.1 弱体化 (Debuffs)
プレイヤーにとって不利な効果をもたらします。

| 名称 | 効果 | 継続条件 / 解除方法 | 備考 |
| :--- | :--- | :--- | :--- |
| **毒 (Poison)** | 一定時間（例: 5秒）ごとに最大HPの一定割合（例: 2%）のダメージを受ける。 | 自然経過（30秒〜60秒）または「毒消し」等のアイテム使用。 | [戦闘システム](Combat-System.md) 参照。 |
| **スタン (Stun)** | 全ての行動（移動、攻撃、アイテム使用）が不能になる。 | 一定時間の経過（例: 3秒）。 | 攻撃を受けた際やトラップで発生。 |
| **鈍足 (Slow)** | 攻撃インターバルおよび移動速度（クールタイム）が 2 倍になる。 | 一定時間の経過、または特定の魔法・アイテム。 | [戦闘システム](Combat-System.md) 参照。 |
| **呪い (Curse)** | 装備しているアイテムが固定され、通常の操作で「外す」ことができなくなる。 | 「解呪」の巻物などの特殊な手段。 | [装備システム](Equipment-System.md) 参照。 |
| **空腹 (Hungry)** | 自然回復（HP/スタミナ）が停止する。 | 満腹度を 20% 以上にする。 | [満腹度システム](Hunger-System.md) 参照。 |
| **飢餓 (Starving)** | 自然回復が停止し、5秒ごとに最大HPの 1% のダメージを受ける。 | 満腹度を 1% 以上にする。 | [満腹度システム](Hunger-System.md) 参照。 |

### 2.2 強化 (Buffs)
プレイヤーにとって有利な効果をもたらします。

| 名称 | 効果 | 継続条件 / 解除方法 | 備考 |
| :--- | :--- | :--- | :--- |
| **再生 (Regeneration)** | 自然回復速度が向上する。 | 一定時間の経過、または特定のアイテム使用。 | |
| **満腹 (Satiated)** | HPとスタミナの自然回復速度が 1.2 倍になる。 | 満腹度が 80% 以上の間継続。 | [満腹度システム](Hunger-System.md) 参照。 |

## 3. 技術的実装

### 3.1 データ構造
`Player` モデル内の `statusEffects: string[]` 配列で管理されます。

```typescript
// 例: 現在のプレイヤーの状態
{
  id: "player-123",
  // ...
  statusEffects: ["Poison", "Slow"], // 毒と鈍足状態
}
```

### 3.2 判定ロジック
- **継続ダメージ (DoT)**: `IntervalService` 等の定期的なティック処理の中で、`statusEffects` に `Poison` が含まれているか判定し、ダメージ計算を行います。
- **行動制限**: 攻撃や移動のコマンドを送信する前に、`Stun` 等の状態が含まれていないかクライアント側でチェックし、サーバー側でも最終的なバリデーションを行います。
- **ステータス補正**: 攻撃インターバルや回避率の算出時に、`Slow` や `Regeneration` の有無を確認し、計算式に倍率を適用します。

## 4. UI・UX 表示
詳細は [UI・UX設計](UI-UX-Design.md) を参照してください。

- **ステータスバー**: `StatusBarComponent` において、現在の状態異常をアイコンまたはテキストで一覧表示します。
- **ログ出力**: 状態異常が発生・解除された際、`MessageComponent` を通じてプレイヤーに通知します。
  - 例: 「毒を受けてしまった！」、「体のしびれが取れた。」
