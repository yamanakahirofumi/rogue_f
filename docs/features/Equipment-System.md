# Equipment-System

## 1. 概要
装備システムは、プレイヤーが武器、防具、装飾品を身に着けることでステータスを強化し、ダンジョン探索や戦闘を有利に進めるための仕組みです。インベントリ内のアイテムを「装備」することで、そのアイテムが持つ補正値がプレイヤーの能力に反映されます。

## 2. 装備スロット
プレイヤーは以下のスロットに、それぞれ 1 つずつアイテムを装備できます。

- **武器 (Weapon)**: 主に攻撃力を上昇させます。
- **防具 (Armor)**: 主に防御力を上昇させます。
- **装飾品 (Accessory)**: 攻撃力、防御力、敏捷性、器用さ、素早さなど、様々なステータスを補助します。

## 3. 装備のルール

### 3.1 装備 (Equip)
- インベントリ内にある `subType` が `weapon`, `armor`, `accessory` のいずれかであるアイテムを選択し、「装備する」アクションを行うことで装備状態になります。
- 既に同じスロットに他のアイテムを装備している場合、自動的に古い装備が外され（インベントリに戻り）、新しいアイテムが装備されます。

### 3.2 装備解除 (Unequip)
- 現在装備しているアイテムを選択し、「外す」アクションを行うことで装備を解除できます。
- 解除されたアイテムはインベントリに戻ります。インベントリが満杯の場合、装備を外すことはできません。

### 3.3 呪いと装備 (Curses)
- [アイテム識別システム](Item-Identification-System.md) に基づき、呪われた（`isCursed: true`）装備品を装備すると、通常の「外す」アクションが使用不能になります。
- 呪いを解く（解呪の巻物などを使用する）か、特定のイベントが発生しない限り、その装備を固定されます。

## 4. ステータスへの影響
装備品は装着中、以下のプレイヤーパラメータ（[Playerデータモデル](../implementation/Implementation-Details.md) 参照）に補正を加えます。

- **攻撃力 (attack)**: 武器および一部の装飾品によって加算されます。
- **防御力 (defense)**: 防具および一部の装飾品によって加算されます。
- **敏捷性 (agility)**: [戦闘システム](Combat-System.md) における攻撃インターバルを短縮します。
- **器用さ (dexterity)**: 命中率を向上させます。
- **素早さ (speed)**: 回避率を向上させます。

補正の計算例:
`現在の攻撃力 = プレイヤーの基本攻撃力 + 装備中の武器の補正値 + 装備中の装飾品の補正値`

## 5. データモデルとの関連
本システムは `InventoryItem` インターフェースの拡張によって管理されます。

### 5.1 装備品に必要なプロパティ
`InventoryItem` に以下の追加情報を（メタデータとして）保持することを想定します。

- `subType`: `weapon`, `armor`, `accessory`
- `attackBonus`: 攻撃力補正値
- `defenseBonus`: 防御力補正値
- `agilityBonus`: 敏捷性補正値
- `dexterityBonus`: 器用さ補正値
- `speedBonus`: 素早さ補正値

### 5.2 プレイヤーの状態
`Player` オブジェクトには、現在装備中のアイテムの ID を保持する以下のフィールドが含まれます。
- `weaponId`: 装備中の武器の ID
- `armorId`: 装備中の防具の ID
- `accessoryId`: 装備中の装飾品の ID
