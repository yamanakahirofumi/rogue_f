# UI-UX-Design

## 1. 画面構成とルーティング
アプリケーションは以下のルートで構成されています。

- `/user/create`: **ユーザー作成画面**
  - 新規プレイヤーの作成を行います。
- `/play`: **プレイ画面**
  - ダンジョン探索のメイン画面です。
- `/admin`: **管理画面**
  - 世界設定、アイテム設定、メニュー管理などを行います。
  - `/admin/menu`: 管理メニューを表示します。
  - `/admin/menu/menu`: 世界設定（`WorldComponent`）を表示します（子ルート）。

## 2. コンポーネント階層
主なコンポーネントの階層構造は以下の通りです。

- `AppComponent`
  - `CreateUserComponent` (ユーザー作成)
  - `DungeonComponent` (プレイ画面)
    - `MessageComponent` (メッセージ表示)
    - `StatusBarComponent` (ステータス表示)
  - `AdminModule` (遅延読み込み)
    - `MenuComponent` (管理メニュー)
      - `WorldComponent` (世界設定、子ルートとして表示)

## 3. 基本デザイン
- **背景色**: 黒 (`background: black`)
- **文字色**: 白 (`color: white`)
- **フォント**: `"Courier New"`, `"Noto Sans Mono CJK"`, monospace (ローグライクらしいレトロな雰囲気を演出)
- **操作フィードバック**:
  - ゲーム内メッセージ（`MessageComponent`）によるイベントログ出力。
  - ステータスバー（`StatusBarComponent`）によるリアルタイムな状態表示。
    - **HP/スタミナ**: プログレスバーで視覚的に表示。
    - **満腹度**: アイコンの色変化や数値で空腹状態を警告。
    - **状態異常**: 付与されているバフ・デバフのアイコンを表示し、ホバーで詳細を確認可能。

## 4. 管理者用 UI (Admin UI)
管理画面は、世界の構築と運営を効率的に行うための専用インターフェースを提供します。

### 4.1 メニュー構成
- **Dungeon タブ**:
    - ダンジョンの新規作成・編集。
    - リアルタイムマップ上でのモンスター・トラップのドラッグ＆ドロップ配置（検討中）。
- **Item タブ**:
    - 倉庫内のアイテム一覧表示、ソート、フィルタリング。
    - ショップへの陳列アイテム選択と価格設定。
- **World タブ**:
    - 接続中の他サーバー一覧と信頼関係の状態表示。
    - 世界の基本ルール（PVPのON/OFF等）のトグルスイッチ。

### 4.2 操作フィードバック
- 管理アクション（配置、価格変更等）の成功・失敗をトースト通知やログで即座に通知。
- リアルタイムで変動する「基本市場価格」のグラフ表示（検討中）。
