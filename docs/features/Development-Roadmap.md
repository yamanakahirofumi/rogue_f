## 3. 今後の課題（実装優先順位）
1. **バックエンドモデルとの同期**: 策定された [Playerデータモデル](../implementation/Implementation-Details.md) に基づき、HP、スタミナ、経験値などの属性をバックエンドから取得・反映する。
2. **エラーハンドリングの強化**: APIエラーやネットワーク切断（SSE再接続など）への対応。
3. **ゲームルールの実装**: [戦闘システム](Combat-System.md) および [レベルアップシステム](Leveling-System.md) のロジック実装。
4. **管理画面の機能拡充**: `admin` 配下のコンポーネントにおける「ダンジョン作成」「モンスター配置」「アイテム屋経営」などの管理者向け機能の実装。
5. **マルチプレイヤー同期**: 複数人での同時プレイを支えるリアルタイム同期機能の強化。
6. **世界間連携システム**: 他の世界の管理者との合意プロセスおよび、サーバー間（または世界間）の移動機能の実装。

## 4. 完了済みの課題 (History)

### 2026年3月
- **仕様の明確化とステータス定義の追加**: `luck`（運）ステータスの追加、レベルアップ時の成長法則（HP/運）の確定、トラップ探索・解除の計算式策定、およびPK敗北時のペナルティ内容の具体化。
- **ドキュメント間のデータモデル不整合の修正**: 満腹度、戦闘ステータス、インベントリ、状態異常の欠落および Angular/TypeScript バージョン表記の不整合を修正。
- **ユニットテストの安定化**: `HttpClientModule` や `Router` の欠落、タイトル不一致などの問題を修正し、テストが正常にパスするように改善。
- **仕様の具体化と欠落情報の追加**: モンスター形態の成長システム、ダンジョン建設容量の計算式、および各種システムの曖昧な数値（毒ダメージ、AI視界、ショップ価格帯等）を確定。
- **アイテム・数値仕様の補完**: 全アイテムのマスターリスト作成、繁殖コストと孵化時間の確定、およびモンスター形態への `luck` ステータスの反映。
