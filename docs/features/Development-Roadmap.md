# Development-Roadmap

## 1. 開発フェーズと目標
現状は基本機能の実装フェーズ（MVPフェーズ）にあります。

## 2. 現在の開発状況と課題

### 2.1 既知のバグ・不整合
- **解消済み**: ドキュメント間のデータモデル不整合（満腹度、戦闘ステータス、インベントリ、状態異常の欠落）および Angular バージョン表記の誤りは修正されました。

### 2.2 テスト状況
現在、以下の理由により複数の単体テストが失敗しています：
- **HttpClientModuleの欠落**: `HttpClientTestingModule` がインポートされていない。
- **Routerの欠落**: `RouterTestingModule` がインポートされていない。
- **期待値の不一致**: `AppComponent` のタイトル検証（`rogueF` vs `rogue-f`）。
- **DOM要素の不在**: `AppComponent` で存在しない `.content span` を探している。
- **アニメーションモジュールの欠落**: `BrowserAnimationsModule` が必要。

## 3. 今後の課題（実装優先順位）
1. **バックエンドモデルとの同期**: 策定された [Playerデータモデル](../implementation/Implementation-Details.md) に基づき、HP、スタミナ、経験値などの属性をバックエンドから取得・反映する。
2. **エラーハンドリングの強化**: APIエラーやネットワーク切断（SSE再接続など）への対応。
3. **ゲームルールの実装**: [戦闘システム](Combat-System.md) および [レベルアップシステム](Leveling-System.md) のロジック実装。
4. **管理画面の機能拡充**: `admin` 配下のコンポーネントにおける「ダンジョン作成」「モンスター配置」「アイテム屋経営」などの管理者向け機能の実装。
5. **マルチプレイヤー同期**: 複数人での同時プレイを支えるリアルタイム同期機能の強化。
6. **世界間連携システム**: 他の世界の管理者との合意プロセスおよび、サーバー間（または世界間）の移動機能の実装。
