# TODO-Details

本ドキュメントでは、RogueF の面白さを向上させるための機能追加や技術的な課題を管理します。

## 1. ゲームデザイン面

### [ ] 戦闘システムの実装
- **概要**: モンスターとの戦闘。
- **検討内容**:
    - 時間ベース（リアルタイム/アクティブタイム制）のバトルの詳細。
    - 攻撃速度やクールタイムの導入。
    - ダメージ計算式（攻撃力、防御力、乱数）。
    - 敵のAIパターン（執拗に追いかけてくる、距離を取る、ランダム移動）。

### [ ] 経験値とレベルアップ
- **概要**: プレイヤーの成長要素による長期的な目標設定。
- **検討内容**:
    - 経験値の獲得源（敵を倒す、階層を進む）。
    - レベルアップ時のステータス上昇量。
    - スキルツリーや職業（クラス）の導入。

### [ ] アイテム識別システム
- **概要**: 未識別のアイテムを使用・鑑定する楽しみとリスク。
- **検討内容**:
    - 識別方法（鑑定の巻物、使用して判明、装備して判明）。
    - ランダムな効果のバリエーション（呪われた装備、祝福されたアイテム）。

### [ ] 満腹度（食料）システム
- **概要**: 探索のリソース管理の深化。
- **検討内容**:
    - 空腹時のデバフ（HP減少速度の上昇など）。
    - 食料アイテムの入手頻度とバランス。

## 2. 技術面

### [ ] セーブ・ロード機能
- **概要**: プレイ状況を保存し、中断・再開できるようにする。
- **検討内容**:
    - ローカルストレージまたはバックエンドDBへの保存タイミング。
    - オートセーブの実装。
    - パーマデス（死んだらデータ消去）との兼ね合い。

### [ ] ダンジョンの動的生成アルゴリズムの改良
- **概要**: 毎回異なる、かつ探索しがいのあるマップ生成。
- **検討内容**:
    - 部屋と通路の配置アルゴリズム（BSP、ランダムウォーク等）。
    - 特殊な部屋（店、宝部屋、中ボス部屋）の生成条件。

## 3. アセット・リソース面

### [ ] タイルセットの拡張
- **概要**: 視覚的な多様性による飽きの防止。
- **検討内容**:
    - バイオーム別の背景（洞窟、森、氷、溶岩）。
    - モンスターやアイテムの個別のスプライト。

### [ ] サウンドエフェクト (SE) と BGM
- **概要**: 没入感の向上。
- **検討内容**:
    - 移動音、攻撃音、アイテム取得音、レベルアップ音。
    - 階層や状況に応じたBGMの切り替え。

## 4. 実装時の詳細事項

### [ ] ミニマップ機能
- **概要**: 現在位置と探索済みの場所を把握しやすくする。
- **検討内容**:
    - 画面の隅に表示する小さなマップ。
    - 探索済みエリアと未探索エリア（フォグ・オブ・ウォー）の可視化。

## 5. 人間が主導すべき設計・調整面

### [ ] ゲームバランスの微調整
- **概要**: 「難しすぎず、易しすぎない」手応えのあるバランス。
- **検討内容**:
    - **人間が考えるべきこと**: 階層ごとの敵の強さのカーブ、アイテムの出現率、プレイヤーが理不尽さを感じない難易度設計。

### [ ] ストーリー・世界観の構築
- **概要**: 探索の目的を強化し、プレイヤーを引き込む。
- **検討内容**:
    - **人間が考えるべきこと**: ダンジョンの最深部に何があるのか、NPCのセリフ、背景設定（Lore）の断片。
